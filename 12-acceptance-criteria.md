# Acceptance Criteria

## User Registration

* User can enter required registration information.
* User account is created successfully.
* User receives confirmation of successful registration.

## User Login

* User can enter valid credentials.
* System authenticates the user successfully.
* User is redirected to the dashboard after login.

## Create Task

* User can enter a task title.
* User can add a task description.
* User can set a deadline.
* Task is saved successfully and appears in the task list.

## Edit Task

* User can modify task information.
* Updated information is saved successfully.
* Changes are reflected in the task list.

## Delete Task

* User can delete an existing task.
* Deleted task is removed from the task list.

## Mark Task as Completed

* User can mark a task as completed.
* Task status is updated successfully.
* Completed task is reflected in progress tracking.

## Deadline Tracking

* User can assign deadlines to tasks.
* Upcoming deadlines are displayed on the dashboard.
* Overdue tasks are clearly identified.

## Calendar View

* Tasks and deadlines are displayed on the calendar.
* User can navigate between calendar dates.
* Calendar data updates automatically when tasks are modified.

## Study Session Scheduling

* User can create study sessions.
* User can specify date and time for each session.
* Scheduled sessions appear on the calendar.

## Notifications

* User receives reminders before task deadlines.
* User receives reminders before scheduled study sessions.
* Notifications contain relevant task information.

## Progress Tracking

* Completed tasks are counted correctly.
* Progress statistics are updated automatically.
* User can view progress information from the dashboard.
